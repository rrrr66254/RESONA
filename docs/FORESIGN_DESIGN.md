# ForeSign — 설계 문서 v0.3

**Part-Structured Latent-Predictive Pretraining for Multilingual Lexical Sign Representation**

> 상태: 설계 단계 (코드 없음). v0.3, 2026-06-10.
> v0.1 → v0.2: 3인 모의 심사(R1: SSL/JEPA, R2: 수화언어학/윤리, R3: 시니어 AC) 반영.
> v0.2 → v0.3: 재심(rebuttal round) 반영. 제목에서 "Phonologically-Structured"를 철회
> (§6에서 음운론적 해석을 "검증 대상"으로 강등했으므로 제목에 쓰면 자기모순 — R2 재심)
> 하고 "Lexical" 스코프를 명시. 변경 대조표는 §14.

## 0. 한 문단 요약

ForeSign의 중심은 시스템이 아니라 **과학적 질문**이다:

> **"수화의 음운 구조(수지/비수지 채널 분해, 양손 대칭 제약, 시간적 선행)는
> 일반(generic) 멀티블록 마스킹보다 나은 자기지도 마스킹 prior인가?"**

이를 검증하기 위해 11개 수화언어·89.5만 클립의 2D 포즈(RESONA-77) 위에서
JEPA식 잠재 예측 사전학습을 수행하되, 마스킹 가족(M1 부위 구조 / M2 선행 예측 /
M3 V-JEPA 멀티블록)을 유일한 독립변수로 통제 비교하고, 학습된 표현이 실제로
음운 정보를 담는지를 **ASL-LEX 음운 파라미터 프로브**(손모양/위치/움직임 디코딩)로
직접 측정한다. 부가 기여: ① 언어 간 전이 분석(저자원 수화 포함, iconicity 통제),
② dataset-ID/language-ID/signer-ID 진단 프로브 3종. S-JEPA 첫 공개 재구현은
기여 목록이 아닌 부산물로만 언급(§3과 일관 — v0.2의 §0/§3 모순 제거, R3 재심).
주장하지 않는 것: world model, anticipation "능력", SLT SOTA, 프라이버시 해결(§1.2).

## 1. 주장 규율 (Claims Discipline)

### 1.1 주장하는 것 (실험으로 직접 지지됨)

| # | 주장 | 지지 실험 |
|---|---|---|
| C1 | 수화 포즈에 대한 첫 JEPA식(연속 잠재 예측 + EMA 타깃) 사전학습 ("sign+JEPA" arXiv 0건, 2026-06 검증) | 베이스라인 통제 비교 (§8.7) |
| C2 | 부위 구조/선행 마스킹이 일반 멀티블록 대비 표현 품질에 주는 효과의 정량화 (**결과가 null이어도 보고** — 사전등록형 ablation) | M1/M2/M3 스위치 ablation (§6) |
| C3 | 학습 표현의 음운 파라미터 부호화 정도 | ASL-LEX 프로브 (§8.2) |
| C4 | 멀티링구얼 사전학습 코퍼스 구성이 저자원 전이에 주는 효과 — KSL은 언어 전이를 부분 분리, 단일 데이터셋 언어(TR/RSL)는 "데이터셋 전이"로 한정 표기 (R3 재심: C4 문구에 한정어 내장) | 코퍼스 ablation × 전이 (§8.4) |
| C5 | 부분 관찰 조건의 조기 인식(early recognition) 특성 분석 | §8.3 (벤치마크가 아닌 분석) |

### 1.2 주장하지 않는 것 (v0.1에서 삭제·강등된 표현)

- ~~"world model"~~ — 롤아웃/계획 실험이 없으므로 전면 삭제 (R3)
- ~~"anticipation capability"~~ — 측정되는 것은 부분 관찰 분류 강건성이므로
  "early recognition"으로 개명; "anticipation"은 M2 마스킹 목적함수의 이름으로만 사용 (R3)
- ~~"Sign Anticipation Benchmark"(헤드라인 기여)~~ — 신규 데이터·주석이 없는
  프로토콜이므로 분석 섹션으로 강등; early action recognition 문헌(Ryoo 2011~,
  survey arXiv:2107.05140)을 명시 인용하고 그 프로토콜의 수화 적용임을 밝힘 (R1·R3)
- ~~"privacy-by-design"(기여 주장)~~ — 포즈는 얼굴 픽셀 제거라는 **좁은** 완화일 뿐이며
  포즈 시퀀스도 재식별 가능함을 명시; signer-ID 프로브(§8.5)로 정량화하고
  서론 한 문장으로 제한 (R2·R3)
- ~~"수화 이해(understanding)"~~ — 코퍼스가 사전형 위주이므로 "수화 어휘 표현
  (lexical sign representation)"으로 스코프 명시; 문장 데이터 기여 비율을
  토큰 단위로 보고 (R2 W1)

## 2. 관련 연구와 신규성 위치 (v0.1 누락분 보강)

v0.1이 누락했던 선행연구를 포함한 전체 델타 표:

| 선행연구 | 무엇을 했나 | ForeSign과의 차이 |
|---|---|---|
| **SignBERT (ICCV'21) / SignBERT+ (TPAMI'23)** | 수화 손 포즈 SSL, 다층(관절/프레임/클립) 마스킹 + 손 모델 prior 디코더 | **좌표/prior 복원 vs 우리는 잠재 예측**; 손 중심 vs 전신 4채널; 단일/소수 언어. **베이스라인으로 직접 비교** |
| **SignCLIP (EMNLP'24)** | Spreadthesign 44개 언어 포즈-텍스트 contrastive | 텍스트 지도 vs 우리는 무지도; 단일 출처(사전) vs 22개 출처 다도메인. 교차언어 실험의 직접 선행 — 인용·비교 |
| **OpenHands (2021)** | 포즈 SSL + 교차언어 전이 (저자원) | 교차언어 전이 "최초" 주장 불가 — 우리는 스케일업 + 코퍼스 구성 ablation으로 차별화 |
| **MASA (2024)** | 모션 인지 마스크드 AE (수화) | 복원 계열. 베이스라인 후보 |
| **SignRep (CVPR'25)** | 스켈레톤 prior 기반 마스크드 수화 표현 | 복원 계열 + prior. 인용·비교 |
| **SHuBERT (ACL'25)** | 비디오 crop 3스트림, 클러스터 예측, ASL 1,000h | 이산 클러스터 vs 연속 잠재; 비디오 vs 포즈; 단일 언어. 참고 비교(모달리티 명시) |
| **Sigma / SSL-SLR (2025-26)** | 스켈레톤 수화 SSL (의미 정렬/복원) | 포즈 트랙 직접 경쟁 — 공개 수치 비교 |
| **Uni-Sign (ICLR'25)** | 지도 생성형 통합 사전학습 (CSL 1,985h) | 지도 vs 무지도 |
| **S-JEPA (ECCV'24)** | 스켈레톤(3D Kinect) JEPA, 액션 60-120클래스 | 도메인(언어 vs 액션), 2D+conf, 멀티링구얼, 마스킹 설계. **원형 이식을 베이스라인으로** |
| **V-JEPA / V-JEPA 2** | 비디오 JEPA 레시피 | 레시피 차용을 명시 — 기여는 레시피가 아니라 마스킹 prior 질문(C2) |
| **SignVerse-2M (2026.5)** | 2M 클립·55개 언어 통합 2D 포즈 코퍼스 | 규모 주장 포기. 관련연구에 즉시 인용 + **서브셋 추가 사전학습 실험으로 위협을 강건성 결과로 전환** (§4.4, R3 권고) |
| 조기 행동 인식 (Ryoo'11~, 서베이 2107.05140) | 관찰비율 프로토콜 표준 | §8.3은 이 프로토콜의 수화 적용임을 명시 — "신규 벤치마크" 주장 안 함 |
| Signing Activity Projection (2606.09424) | DGS 상호작용 턴테이킹 예측 | "수화 anticipation 최초" 주장 불가 — 어휘 수준 조기 인식으로 구분 |

**잔존 신규성 (정직한 진술)**: ① 연속 잠재 예측(JEPA) 패러다임의 수화 첫 적용(C1),
② 마스킹 prior에 대한 사전등록형 통제 실험(C2)과 음운 프로브(C3)라는 **과학적
질문 중심 설계** — 시스템 신규성이 아니라 발견(finding)이 기여다. M1/M2가
M3 대비 유의미하면 그것은 수화 구조에 관한 주장이고, null이면 "일반 마스킹으로
충분하다"는 역시 보고 가치가 있는 결과다 (사전등록이 이를 보호).

## 3. S-JEPA / V-JEPA 정독 요약 (v0.1과 동일 — 핵심만)

- S-JEPA(ECCV 2024): 공개 코드·arXiv 없음 → MAMP 코드 + ECVA 전문에서 재구성.
  EMA 타깃 필수(제거 시 1.6%로 붕괴), **타깃은 풀 시퀀스를 본 타깃 인코더의 출력에서
  선택**(+7.2pp), CE+centering 손실, 다운스트림은 EMA 인코더.
- V-JEPA: **시간축 전체를 덮는 멀티블록**(랜덤 튜브 대비 >20pt), L1 latent 손실,
  predictor 좁게(d/2), attentive probe. V-JEPA 2: warmup–constant–decay + EMA/wd 고정
  (anytime checkpoint), progressive length cooldown.
- 채택: V-JEPA 마스킹 기하 + L1 기본, S-JEPA CE+centering fallback.
- 재구성 리스크 완화: S-JEPA 저자(EPFL) 이메일 문의 병행, 재구성임을 논문에 명시.
  "첫 공개 스켈레톤 JEPA 구현"은 기여 목록이 아닌 부산물로만 언급 (R2 W10c).

## 4. 데이터 계층

### 4.1 코퍼스 구성과 정직한 기술 (R2 W1)

RESONA-77 H5: 22개 데이터셋, 11개 언어, 895,427클립. **사전형(고립 단어) 클립이
다수, ASL 70%** — 이를 숨기지 않고 다음을 본문에 보고한다:

- temperature 샘플링 후 **토큰 단위** 문장/사전형 기여 비율 (R2 Q1)
- 데이터셋별 데이터 스테이트먼트(출처, 수집 방식, 라이선스, 동의 형태) — §10
- 스코프 결론: 본 연구의 표현은 "어휘 수준 수화 표현"이며 담화 수준 주장은 하지 않음

### 4.2 패딩·confidence 명세 (R1 W4, R2 W10b/Q8)

- 패딩 프레임은 **conf=0으로 명시 기록**, valid_mask와 함께 이중 표시
- **패딩 토큰은 (a) 마스킹 통계, (b) 예측 타깃, (c) 손실 분모에서 전부 제외**
- 사전형 클립의 **유효 토큰 기준 실효 마스킹률**을 로깅·보고 (R1 Q7)
- 토큰 평균 conf < 0.2 → 타깃 제외; conf-가중 모션 saliency m = min(c_t, c_{t−1})·|Δx|

### 4.3 우세손(handedness) 문제 (R2 Q2)

22개 코퍼스 모두 signer 우세손 주석이 없다. 따라서 v0.1의 "비우세손 마스킹
(Battison)" 프레이밍을 **철회**하고:

- 클립별 **모션 에너지 기반 활동손 추정** (Σ min(c)·|Δx| per hand)으로
  활동손/보조손을 조작적으로 정의 — 이는 음운론 주장이 아닌 휴리스틱임을 명시
- 한손 마스킹(M1c)은 **양손 활동 클립에서만** 적용 (한손 수화에서 휴지손 마스킹은
  무의미한 신호이므로)
- **휴리스틱 검증 (v0.3)**: ASL-LEX Sign Type(한손/대칭/비대칭) 코딩을 ground
  truth로 활동손 추정의 혼동행렬 보고 — 비대칭 양손 수화(베이스손이 정지하되
  음운적으로 대조적)에서 모션 에너지가 오분류하는 비율을 정량화 (R2 재심 N7)

### 4.4 SignVerse-2M 대응 (R3)

- 관련연구 인용 + 라이선스 확인 후 **서브셋(예: 200K클립) 추가 사전학습 실험** 1건:
  "외부 코퍼스로 코퍼스를 교체해도 마스킹 효과(C2)가 재현되는가" — 위협을
  강건성 증거로 전환. 라이선스 불가 시 한계로 명시.

### 4.5 View 증강 (v0.1 유지)

2D affine + temporal 리샘플 + jitter. horizontal flip 기본 OFF (거울상 손모양은
음운적으로 별개) — 사용 시 L/R 그룹·관절 스왑 동반, ablation 전용.

## 5. 모델

### 5.1 토큰화 — body 재구조화로 M1a 손목 누설 제거 (R1 W2)

v0.1의 결함: body 토큰에 손목이 포함되어 "손 마스킹" 시 위치·움직임이 누설.
v0.2는 body를 3토큰으로 분해:

| 부위 | 공간 토큰 | 구성 |
|---|---|---|
| trunk/head | 1 | 코, 양 힙 (+어깨는 L/R arm에 귀속) |
| L-arm / R-arm | 2 | 각 {어깨, 팔꿈치, **손목**} |
| lhand / rhand | 12 | 손바닥 1 + 손가락 5 (각 손 6) |
| face | 3 | mouth8 / 눈·눈썹 / 윤곽 |
| **계** | **18** | T=128, l=4 → 18×32 = **576 토큰** |

temporal RoPE + learnable spatial embed, linear patchify (GCN stem은 ablation),
part-validity는 learnable 벡터 가산 — v0.1과 동일.

### 5.2 인코더/Predictor/손실 (v0.1 유지)

S(d384/12L) → B(d768/12L) → L(게이트). EMA 타깃(0.999 고정 시작), 타깃은 풀 시퀀스
출력에서 선택, predictor 좁은 transformer + zero-init 마스크 토큰. **L1 latent 기본,
CE+centering fallback.** 붕괴 모니터링(유효랭크·특이값·분산) 상시 로깅.
"V-JEPA의 무붕괴는 픽셀 비디오에서의 증거이며 234차원 포즈로의 전이는 가정"임을
명시하고 S 스케일 조기 검증으로 관리 (R1 W8b).

## 6. 마스킹 설계 v2 — 사전등록형 통제 실험 (C2)

모든 마스크는 유효(비패딩) 토큰 위에서 정의. 가족 선택확률 (0.6 / 0.25 / 0.15) 초기값.

### M3 — V-JEPA 멀티블록 (대조군/기본)

short 8블록×공간 0.15 + long 2블록×공간 0.7, **시간 스케일 1.0**, conf-가중
saliency 바이어스. *이것만으로 학습한 모델이 모든 비교의 대조군.*

**M3′ — 비율·기하 매칭 대조군 (v0.3 신설 — R1 재심 N1, C2의 필수 대조군)**:
M1a와 동일한 마스킹 비율·공간 연속성·시간 범위를 갖되 **해부학적 부위 경계를
무시한** 무작위 연속 토큰 집합을 마스킹. M1 vs M3 비교는 비율·기하가 동시에
달라 해석 불가하므로, **M1 vs M3′ 비교만이 "부위 구조" 효과를 "상관 토큰 대량
마스킹" 효과와 분리한다.** 결정 게이트(§7)의 5조건에 포함.

### M1 — 부위 구조(part-structured) 마스크 — "음운론적"에서 개명 (R2 W2)

명칭을 정직화한다: 마스크는 **해부학적 부위 단위**로 정의되며, 음운론적 해석은
주장이 아니라 **검증 대상**이다(→ §8.2 음운 프로브가 판정).

- **M1a-strict** (0.5): 양손 12토큰 + **양 arm 2토큰** 마스킹 — 위치·움직임·손모양
  전부를 trunk+face 5토큰에서 예측. 비수지→수지 채널 추론의 최대 난도 조건
- **M1a-loose** (0.2): 양손 12토큰만 마스킹, arm(손목) 가시 — 위치는 주고
  손모양·방향만 예측. **strict vs loose 비교가 "무엇이 학습 가능한 신호인가"를
  파라미터 수준(위치 vs 손모양)에서 분해** (R1 W2의 ill-posed 비판에 대한 실험적 응답)
- **M1b** (0.15): face 3토큰 마스킹 (역방향 추론)
- **M1c** (0.15): 양손 활동 클립에서 한 손(hand+arm) 마스킹 — 양손 수화의
  대칭/교대 구조 활용. Battison 인용은 동기 부여로만, 검증은 음운 프로브로

**ill-posed 우려에 대한 명시적 입장** (R1 W2): 다중 정답 조건에서 L1 잠재 예측은
조건부 중앙값으로 수렴할 수 있다. 이것이 표현 학습에 유해한지는 미해결 실험
질문이며, 우리는 (a) M1a 예측의 분산/타깃 분산 비율, (b) face 토큰을 데이터셋 평균
얼굴로 치환해 **재학습한** 대조군 — 평가 시점 치환은 분포 이동과 정보 제거를
혼동하므로 금지 (R2 재심 N4; 성능 무변화 시 비수지 추론 주장 철회),
(c) face 키포인트 conf 통계 보고로 판정한다. **M1a가 작동하지 않는다는 결과도
C2의 유효한 답**이다.

### M2 — 선행(anticipation) 마스크 — 적용 범위 축소 (R1 W3)

- **문장/연속 데이터셋 클립에만 적용** (사전형 클립에서는 onset이 정체를 결정해
  자명하고, 패딩 예측 위험 — R1 지적 수용)
- t₀ ~ U[0.4T, 0.8T] 이후 유효 토큰 전부 마스킹
- **M2 적용 가능 토큰 비율(연속 클립 비중)을 P1 시작 전에 산출·보고** — 비율이
  작으면 M2 ablation은 구조적으로 검정력 부족이므로 게이트 판정 시 그 한계를
  명시 (R2 재심 N8, R1 재심 W3)
- **mean-regression 진단 내장**: 지평별 예측 분산/타깃 분산 비율, 예측-타깃 코사인
  분포를 로깅 — 미래의 다봉성(multimodality)으로 인한 평균 회귀를 정량 보고 (R1 Q3)
- M2=0 ablation이 조기 인식 곡선에 영향이 없으면 "선행 마스킹은 불필요"로
  **정직하게 보고**하고 프레이밍에서 제거 (R1 Q4 — 사전등록이 이 결과도 기여로 만듦)

## 7. 학습 레시피 (v0.1 + 통계 보강)

v0.1 동일(AdamW, wd 0.05 고정, warmup–constant–decay, EMA 0.999, bf16, S→B→L,
Stage 2 문장 cooldown). 추가:

- **프로브/평가는 전부 3-seed + 표준편차 보고** (사전학습은 1-seed임을 명시; R3 §5)
- **결정 게이트 (가장 중요 — v0.3 통계 강화, R3 재심)**: S 스케일에서
  {M3-only, **M3′**, M3+M1, M3+M2, M3+M1+M2} **5조건 × 사전학습 ≥3 seed**를
  최우선 실행. 1-seed 게이트는 "노이즈 수준"이 정의 불가능해 판정 자체가
  노이즈라는 지적 수용. **사전 명시 판정 규칙**: macro-ISLR 및 음운 프로브
  델타 > 2 × pooled seed-SD이면 양성. 규칙·조건·지표를 **P1 시작 전 OSF
  (또는 arXiv 테크노트)에 타임스탬프 공개** — 등록 메커니즘 없는 "사전등록"
  서술은 post-hoc으로 간주됨. null이면 교차언어 스토리(C4) 중심 재조준 +
  부정 결과 보고. **추가 게이트 조건(R-8)**: JEPA 계열이 복원 베이스라인
  (MSM/SignBERT+류) 이상일 것 — 미달 시 잠재+복원 하이브리드 손실 ablation으로
  전환하고 결과를 그대로 보고

## 8. 평가 스위트 v2

### 8.1 ISLR 프로브 (v0.1 유지 + 보강)

attentive/linear/kNN, signer-independent 분할(자체 분할은 데이터셋별 문서화 — R2 W8),
언어 macro 평균, 3-seed. **ISLR은 표현 품질의 대리 지표이며 "이해"의 증거가 아님을
명시** (R2 W9).

### 8.2 음운 파라미터 프로브 — 신규, C3의 직접 검증 (R2 W2 처방 수용)

- **ASL-LEX 2.0 / Sem-Lex의 음운 코딩**(손모양, 주 위치, 움직임 유형, 양손성 등)을
  asl_citizen·semlex 어휘에 매핑 → frozen 표현에서 각 파라미터를 linear probe로 디코딩
- 마스킹 가족별(M3-only vs +M1 vs +M2) 파라미터 디코딩 정확도 비교 —
  **"부위 마스킹이 음운 정보를 더 부호화하게 하는가"를 직접 측정**
- **해석 장치 (v0.3)**: raw 포즈 통계 linear probe와 random-init 인코더 프로브를
  바닥(floor), 지도 SL-GCN 파라미터 인식(Sem-Lex 보고 ~85%)을 천장(ceiling)으로
  병기 — 바닥 없는 "음운 부호화" 주장은 인용형에서 거의 무의미 (R1 재심 N5,
  R2 재심 N3). **C3 결론은 ASL 인용형에 한정됨을 본문에 명시** (R2 재심 N2)
- 이것이 "음운론적"이라는 단어를 쓸 수 있는 유일한 근거이며, 장식적 언어학
  비판(R2 W2)에 대한 구조적 응답

### 8.3 부분 관찰 조기 인식 분석 (벤치마크에서 분석으로 강등)

- 명칭: ~~Sign Anticipation Benchmark~~ → **"Early Recognition under Partial
  Observation" 분석 섹션**. early action recognition 프로토콜(관찰비율 25/50/75/100%)의
  수화 적용임을 명시 인용 (R1 W5, R3)
- **인용형 전이동작 교란 통제** (R2 W4 — v0.1의 최대 결함):
  (a) 모션 onset 검출로 휴지→조음위치 전이 구간을 추정·절단한 곡선을 병행 보고
  — 검출기 자체를 **~200클립 인간 주석 onset 세트**로 정밀도 검증 후 사용,
  주석은 §10.3의 유창한 signer 검토와 연계 (R2 재심),
  (b) 전이 구간 포함/제외 격차 자체를 분석 대상으로,
  (c) 연속 데이터 확장(How2Sign 등 경계 정렬 확보 시)을 후속 단계로 명시 —
  확보 전에는 "고립 수화의 조기 인식"으로만 주장
- per-class onset 정보량 분석은 유지 (세 리뷰어 공통으로 "가장 흥미로운 조각"으로 인정)

### 8.4 교차언어 전이 (C4 — 논문의 제2 기둥)

- 코퍼스 ablation: {ASL-only} vs {ASL+유럽} vs {전체} → KSL/TR/RSL zero/few-shot
- **교란 통제** (R1 W6, R2 W8):
  - KSL은 **2개 독립 데이터셋**(ksl_sign636, ksl_sign103) 보유 — 한쪽을 전이 타깃,
    다른 쪽을 held-out 검증으로 사용해 "언어 전이 vs 캡처 셋업 전이"를 부분 분리
  - 단일 데이터셋 언어(TR/RSL)는 "데이터셋 전이"로 정직하게 명명
  - **iconicity 통제**: ASL-LEX iconicity 노름으로 도상적/자의적 어휘 서브셋을 나눠
    전이 성능을 분리 보고 (도상성 공유에 의한 전이 인플레 차단). 단 ASL-LEX
    도상성은 ASL 기준·비수화자 평정이므로 교차언어 대용물임을 명시하고,
    **~100개 개념 매칭 서브셋**(KSL 번역 등가어의 형태 유사성 수동 평정 —
    §10.3 협력자 검토)으로 보강 (R1 재심 N4, R2 재심 N6)
- OpenHands를 선행으로 인용, 차이는 규모+코퍼스 구성 ablation+통제

### 8.5 진단 프로브 3종 (v0.1의 2종 + signer-ID)

dataset-ID(낮을수록), language-ID(높을수록), **signer-ID probe 신규** —
"포즈도 재식별 가능"을 자인하고 정량화 (privacy 주장을 데이터로 제한; R2 W6, R3).
단일 데이터셋 언어에서 dataset/language 라벨이 일치해 비식별인 문제(R1 W6)는
**다중 데이터셋 언어(ASL 7개, KSL 2개, DGS 4개, LSA 2개)에서만** 두 프로브의
괴리를 해석한다고 명시.

### 8.6 SLT — v1 논문에서 제외 (R2 W5, R3 권고 수용)

frozen 포즈 인코더 + mBART는 비디오 SOTA와 같은 표에서 "지는 판"이므로 v1에서
제거. 재개 조건(둘 다 충족 시 v2): (a) 텍스트 페어 존재 확인(§11 R-1),
(b) 포즈 트랙(Uni-Sign pose 등) 대비 경쟁력 있는 중간 결과. 대신 연속 신호 과제로
**sign spotting/segmentation 프로브를 필수(P2)로 승격** (v0.3) — 인용형 박스를
벗어나는 유일한 연속 신호 증거이며 마스킹·전이 양쪽 스토리의 보험. "선택으로
두면 일정 압박 시 가장 먼저 잘린다"는 R2 재심 지적 수용.

### 8.7 베이스라인 (동일 토큰화·데이터에서 통제 비교)

1. MSM(좌표 복원) 2. **S-JEPA port**(CE+centering 원형) 3. BYOL temporal
4. **SignBERT+ 스타일**(다층 마스킹+복원 — 공개 코드 어댑테이션) 5. **MASA 스타일**
(모션 인지 복원) 6. scratch supervised.
외부 참고 수치: Sigma, SSL-SLR, SignCLIP(교차언어 축), SHuBERT/Uni-Sign(모달리티
명시 참고 — 회피로 보이지 않게 본문에서 격차와 이유를 선제 논의).

## 10. 윤리·라이선스·농인 협력 (신설 — R2 W6/W7)

1. **라이선스 감사 (사전학습 시작 전 완료)**: 22개 코퍼스 전수 — 특히 WLASL
   (연구 전용·재배포 제한), yt_asl(YouTube ToS·signer 무동의), KSL(AI-Hub 파생모델
   조항). 산출물: 코퍼스별 데이터 스테이트먼트 + **가중치 공개 가능성 매트릭스**.
   공개 불가 코퍼스는 사전학습 제외 버전(release-clean run)을 별도 학습.
   **v0.3**: release-clean run은 가중치 공개용에 그치지 않고 — 헤드라인 발견
   (C2/C3)이 release-clean 코퍼스에서 재현되는지를 **본문에** 보고. "동의는 배포가
   아니라 사용의 문제"라는 R2 재심 지적 수용; 미재현 시 한계 절에 명시
2. **프라이버시 주장 축소**: "포즈는 얼굴 픽셀을 제거하지만 signing style은
   생체정보적이며 재식별 가능" — signer-ID 프로브로 정량화(§8.5), 주장은 서론 1문장
3. **농인 협력 (지금 시작)**: (a) 자문 협력자 섭외 — 국내 농인 연구자/한국수어
   통역·교육 기관, 학내 장애학생지원센터 경유, (b) 조기 인식 분석의 per-class
   해석을 유창한 signer와 검증, (c) 논문에 positionality statement와 협력 형태 명시,
   (d) KSL 평가 분할의 커뮤니티 검토. **v0.3 강화 (R2 재심: "하류 검증만 있는
   자문은 협력이 아님")**: 자문 협력자 실명 확보를 **P0 종료 게이트**로 격상,
   자문 보상 예산 명시, §8.3 해석적 주장과 §10 데이터 스테이트먼트에 대한
   협력자 검토·승인(sign-off) 권한 부여. 석사 단독 규모에서 가능한 수준을
   정직하게 기술하고 과장하지 않음
